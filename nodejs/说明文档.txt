一.script加载,放在哪里比较合适
  问题:放在<head>内,当script内容很多时,页面会长时间出现白屏,直到script文件加载完毕                   -----
  	   对应的解决方法:延迟加载,当被用到的时候,再去加载这些script

       放在<body>标签去后,页面虽然不会出现白屏,但是渲染的时候,script文件没有被加载,这些效果是打折扣的-----对应的解决方法:用于显示的script,可以立刻加载,放在<head>内,


二.关于async
   第一步.
   当集合中元素的个数不确定时,而且处理的都是一类方法            ==> 使用有关数组的函数
   当集合中元素的个数确实,而且不是很多时,处理的是不同种类的方法 ==> 使用有关函数数组的数组

   第二步.具体使用何种方法(牢记不论是何种方法,衔接前后之间的函数,都是通过回调函数的)
   如:
   series    :异步函数的同步执行,获得多个结果
   waterfull :异步函数的同步执行,而且前一个函数的返回值,作为后一个函数的参数,获得单个结果
   parallel  :异步函数的异步执行,获得多个结果

   注意:async的回调函数,传入的参数,大致可以分成两类,一类是Boolean型,当有异常时,直接throw err;
        一类是err,则cb(err, ...);


三.有关js,变量和方法的提升(hoisting或hosited),即在函数定义过程中(在编译的时候),会将后面申明的变量的定义,提前放到函数的最前面
   变量提升:如
   	var b = "hello";
	function a(){ 
		alert( b );
		var b = 123;
   	}

   	在编译的过程中,上述变成了:
   	var b = "hello";
   	function a(){	
   		var b ;
   		alert( b );//b等于undefined
   		b = 123;
   	}
